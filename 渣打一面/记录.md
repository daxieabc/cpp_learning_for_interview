先是一些简单的c++基础问题  
1、struct和class区别  
默认访问权限、默认继承权限

2、写一个类时，编译器会默认生成哪些函数  
六种：  
默认构造函数 (Default Constructor)，析构函数 (Destructor)  
拷贝构造函数 (Copy Constructor)，拷贝赋值运算符 (Copy Assignment Operator)  
移动构造函数 (Move Constructor)，移动赋值运算符 (Move Assignment Operator)

3、如何不让编译器生成这些函数  
第一种：声明 = delete（c++ 11）  第二种：声明在private中（老做法）

4、static_cast 和 dynamic_cast 的区别，转换失败会怎样  
功能区别：静态转换和动态转换（编译和运行）  
失败后：static_cast并不会做什么，未定义行为（可以开启编译器选项，让他给出一些场景的警告）
        dynamic_cast，如果转换指针，返回nullptr，引用，抛出std::bad_cast 异常

5、如何生成一个shared_ptr  
使用 std::make_shared  
auto ptr = std::make_shared<MyClass>(10);   
auto ptr_int = std::make_shared<int>(10);

6、malloc和new的区别  
构造函数与析构函数的调用

7、new，operator new，replacement new  
new:分配内存，构造对象  
operator new：只分配内存，可以重载类的operator new，自己分配内存    
replacement new：跳过内存分配，直接在已有内存上构造对象

最后问了perf相关参数的问题：perf stat作用（这个以前确实没用过）  
| 子命令 | 功能描述 | 典型场景 |
| :--- | :--- | :--- |
| `stat` | 统计计数。运行一个命令或附加到进程，统计硬件事件（如周期、指令、缓存缺失）的总数。 | 快速评估程序整体性能，看是否有严重的缓存缺失或分支预测失败。 |
| `record` | 录制采样。运行命令或附加进程，将采样数据（调用栈、时间戳等）保存到 `perf.data` 文件。 | 需要离线详细分析，或者生成火焰图时。 |
| `report` | 报告分析。读取 `perf.data` 文件，以文本或交互式界面展示热点函数、调用图。 | 分析 `record` 生成的数据，找出最耗时的函数。 |
| `top` | 实时监控。类似 `top` 命令，但显示的是函数级别的 CPU 占用率，实时刷新。 | 线上紧急排查，实时查看哪个函数占用了 CPU。 |
| `list` | 列出事件。列出当前系统支持的所有硬件事件和软件事件。 | 查找可用的事件名称（如 `cache-misses`, `context-switches`）。 |
| `trace` | 系统调用追踪。类似 `strace`，但基于内核跟踪点，开销更低，可过滤。 | 分析程序频繁的系统调用、延迟来源。 |
| `annotate` | 代码级汇编分析。在 `report` 基础上，显示具体哪一行汇编代码耗时最多。 | 极致优化，查看具体指令（如内存加载、分支跳转）的开销。 |
| `script` | 脚本处理。将原始采样数据导出为文本或供 Python 脚本处理。 | 自定义数据分析，生成特定格式的报表。 |
| `inject` / `buildid-cache` | 高级工具。用于修改数据文件或管理构建 ID（用于解析符号）。 | 处理容器化环境或剥离符号的二进制文件。 |
